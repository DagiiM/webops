Quickstart

Get up and running with the WebOps API in minutes.

1

Get your API token

Navigate to your WebOps dashboard and generate an API token under Settings → API Keys.

Your API Token 
wps_live_1234567890abcdefghijklmnopqrstuvwxyz
2

Make your first API request

Test your connection by fetching your deployments list.

Request 
curl https://api.webops.dev/v1/deployments \
  -H "Authorization: Token YOUR_API_TOKEN"
Request 
import requests

response = requests.get(
    "https://api.webops.dev/v1/deployments",
    headers={"Authorization": "Token YOUR_API_TOKEN"}
)

print(response.json())
Request 
const response = await fetch('https://api.webops.dev/v1/deployments', {
  headers: {
    'Authorization': 'Token YOUR_API_TOKEN'
  }
});

const data = await response.json();
console.log(data);
Response

You'll receive a JSON array of your deployments with their current status, ports, and metadata.

3

Deploy your first application

Create a new deployment from your GitHub repository.

POST /v1/deployments 
curl -X POST https://api.webops.dev/v1/deployments \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-django-app",
    "repo_url": "https://github.com/username/django-app",
    "branch": "main",
    "env_vars": {
      "DEBUG": "False",
      "SECRET_KEY": "your-secret-key"
    }
  }'

Authentication

Secure your API requests with token-based authentication.

Token Authentication

WebOps uses API tokens to authenticate requests. Include your token in the Authorization header:

Authorization: Token wps_live_1234567890abcdefghijklmnopqrstuvwxyz
Keep your token secure

Never expose your API token in client-side code or public repositories. Treat it like a password.

Generating API Tokens

Create and manage API tokens from your WebOps dashboard:

  1. Navigate to Settings → API Keys
  2. Click Generate New Token
  3. Give your token a descriptive name
  4. Copy and securely store your token (it won't be shown again)

Token Prefixes

wps_live_ Production
wps_test_ Testing

Security Best Practices

  • Rotate tokens regularly
  • Use environment variables
  • Limit token scope when possible
  • Monitor token usage

API Endpoints

Complete reference for all WebOps API endpoints.

Deployments

Manage your application deployments.

GET /v1/deployments
List all deployments

Query Parameters

page optional
Page number for pagination (default: 1)
per_page optional
Results per page, max 100 (default: 20)
status optional
Filter by status: pending, building, running, stopped, failed

Response

200 OK 
{
  "deployments": [
    {
      "id": 1,
      "name": "my-django-app",
      "repo_url": "https://github.com/username/django-app",
      "branch": "main",
      "status": "running",
      "project_type": "django",
      "port": 8001,
      "domain": "myapp.example.com",
      "created_at": "2025-01-15T10:30:00Z",
      "updated_at": "2025-01-15T10:35:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 42,
    "pages": 3
  }
}
POST /v1/deployments
Create a new deployment

Request Body

name required
Unique deployment name (lowercase, alphanumeric, hyphens)
repo_url required
GitHub repository URL (HTTPS)
branch optional
Git branch to deploy (default: "main")
domain optional
Custom domain for deployment
env_vars optional
Object containing environment variables

Example Request

curl -X POST https://api.webops.dev/v1/deployments \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-django-app",
    "repo_url": "https://github.com/username/django-app",
    "branch": "main",
    "domain": "myapp.example.com",
    "env_vars": {
      "DEBUG": "False",
      "SECRET_KEY": "django-secret-key",
      "ALLOWED_HOSTS": "myapp.example.com"
    }
  }'
import requests

data = {
    "name": "my-django-app",
    "repo_url": "https://github.com/username/django-app",
    "branch": "main",
    "domain": "myapp.example.com",
    "env_vars": {
        "DEBUG": "False",
        "SECRET_KEY": "django-secret-key",
        "ALLOWED_HOSTS": "myapp.example.com"
    }
}

response = requests.post(
    "https://api.webops.dev/v1/deployments",
    headers={"Authorization": "Token YOUR_API_TOKEN"},
    json=data
)

print(response.json())
const response = await fetch('https://api.webops.dev/v1/deployments', {
  method: 'POST',
  headers: {
    'Authorization': 'Token YOUR_API_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'my-django-app',
    repo_url: 'https://github.com/username/django-app',
    branch: 'main',
    domain: 'myapp.example.com',
    env_vars: {
      DEBUG: 'False',
      SECRET_KEY: 'django-secret-key',
      ALLOWED_HOSTS: 'myapp.example.com'
    }
  })
});

const data = await response.json();
console.log(data);

Response

201 Created 
{
  "id": 42,
  "name": "my-django-app",
  "status": "pending",
  "message": "Deployment queued successfully"
}

POST /v1/deployments/:name/start
Start a deployment
POST /v1/deployments/:name/stop
Stop a deployment

Delete a deployment

Response

GET /v1/databases/:name
Get database credentials
⚠️
Security Notice

This endpoint returns the decrypted database password. Use with caution and never log or expose credentials.

Response

{
  "name": "my_django_app_db",
  "username": "my_django_app_user",
  "password": "secureP@ssw0rd123",
  "host": "localhost",
  "port": 5432,
  "connection_string": "postgresql://my_django_app_user:secureP@ssw0rd123@localhost:5432/my_django_app_db"

Error Codes

Understanding API error responses.

400

Bad Request

The request was malformed or missing required parameters.

401

Unauthorized

Invalid or missing API token.

403

Forbidden

You don't have permission to access this resource.

404

Not Found

The requested resource doesn't exist.

429

Too Many Requests

You've exceeded the rate limit.

500

Internal Server Error

Something went wrong on our end.

Error Response Format

{
  "error": "Invalid repository URL",
  "message": "Must be a valid GitHub HTTPS URL",
  "status_code": 400
}
